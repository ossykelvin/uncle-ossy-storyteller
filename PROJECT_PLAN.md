## Phase 1 — Make AI generation actually work (½ day)

The single most important gap: only offline placeholders are proven.

1. Add a real provider key to `.env.local`:
   - OpenRouter (recommended, primary): `OPENROUTER_API_KEY` + the 8 `*_MODEL` keys
     (writing/outline/editing/research/continuity/qa/marketing/export).
   - Gemini (fallback): `GEMINI_API_KEY` + `GEMINI_WRITING_MODEL`.
2. Restart, open **Settings → Provider health**, confirm key shows configured.
3. Run the full journey once with a tiny project (1 chapter):
   New Project → Generate Outline → Approve → Generate Chapter → Approve
   (triggers summary + Story Bible update + continuity check) → Export.
4. Fix any real-provider response-shape issues surfaced in `core/ai_client.py`.

**Exit criteria:** a chapter generated by a real model, approved, and exported to DOCX/PDF.

---

## Phase 2 — Security & multi-user hardening (1 day)

Required before exposing to the public internet. Current auth is safe only on localhost.

1. **Per-user salt + stronger hashing.** Replace the static `"local-mvp"` salt in
   `core/auth.py` with a per-user random salt; move to `hashlib.scrypt`/`pbkdf2_hmac`
   (or `passlib`/`bcrypt`). Keep the existing `hmac.compare_digest` check.
2. **Remove default credentials.** Force a password change on first login, or require
   `LOCAL_ADMIN_PASSWORD` to be explicitly set (fail closed if it's still `change-me`).
3. **Project ownership is already enforced** (`list_projects` filters by `owner`) — keep it,
   but confirm direct project-id access in Story Bible/Export pages also checks owner.
4. **Session hardening.** Streamlit session_state resets on refresh; decide if that's
   acceptable or add cookie-based persistence.
5. **Secrets discipline.** Never commit `.env.local`; use `st.secrets` in the cloud.

**Exit criteria:** two separate users can register, log in, and cannot see each other's projects; no default password works.

---

## Phase 3 — Robustness & quality gates (1 day)

1. **`.gitignore`** for `.venv/`, `.env.local`, `data/`, `__pycache__/`, `*.log`.
2. **Pin dependencies** — replace `>=` with `==` in `requirements.txt` for reproducible
   cloud builds (capture the versions just installed). Add `runtime.txt` / `.python-version`
   pinning Python 3.12.
3. **Smoke tests** (pytest): auth round-trip, project create/load/save, each export format
   produces a non-empty file, offline-fallback returns content for every `purpose`.
4. **Export edge cases** — verify DOCX/PDF/EPUB on a project with empty chapters and on
   unicode/emoji content (PDF via reportlab is the fragile one).
5. **Error surfaces** — the UI already wraps generation in try/except; confirm every page
   degrades gracefully when `data/` is empty.

**Exit criteria:** `pytest` green; exports validated; clean repo with no secrets tracked.

---

## Phase 4 — Deploy to the web (½ day)

Path A (fastest, matches README): **Streamlit Community Cloud**

1. Push to a GitHub repo (with `.gitignore` from Phase 3).
2. Create app on Streamlit Cloud pointing at `app.py`.
3. Add all keys under **Manage app → Settings → Secrets** (TOML) — `config.py` already
   reads `st.secrets`.
4. **Storage caveat:** Streamlit Cloud has an *ephemeral* filesystem — `data/*.json`
   resets on restart. For a true multi-user web app, plan the Supabase/Postgres migration
   the README roadmap already anticipates (see Phase 5). For a single-author demo,
   ephemeral storage is acceptable short-term.

Path B (persistent): containerize (`Dockerfile` + the existing `start.bat` logic) and
deploy to Fly.io/Render/Railway with a mounted volume for `data/`.

**Exit criteria:** a public URL where you can log in and generate a chapter.

---

## Phase 5 — Durable backend & feature completion (stretch, 2–4 days)

Turns the demo into a real product. Pull from the README roadmap.

1. **Persistent storage** — migrate `core/storage.py`'s JSON files to Supabase/Postgres so
   data survives restarts and scales to multiple users. Keep the same bundle shape to
   minimize UI churn.
2. **Wire up the orphaned agents** (currently unused): `character_agent`, `dialogue_agent`,
   `research_agent`, `editor_agent`. Add UI entry points or fold them into the writing flow.
3. **Cover image generation** (roadmap item) — `cover_marketing_agent` already does prompts;
   add an image API.
4. Optional: Supabase Auth to replace local auth from Phase 2.

## Decisions made (2026-06-21)

- **Provider:** ✅ **Both** — OpenRouter primary + Gemini fallback. Need keys for each.
- **Audience:** ✅ **Public sign-ups** — strangers register and save real work.
- **Hosting:** ✅ **Streamlit Cloud.**

### What these choices change

> **⚠️ Storage is now the critical-path risk.** Public sign-ups on Streamlit Cloud's
> *ephemeral* filesystem means every user's projects get wiped on each app restart.
> Therefore:
>
> - **Phase 2 (security) is mandatory** — real strangers' credentials and project isolation.
> - **Phase 5's Postgres/Supabase migration is promoted ahead of public launch**, not a
>   stretch goal. Recommended: **Supabase** (free tier, pairs with Streamlit, in the README
>   roadmap, and Supabase Auth can replace local auth from Phase 2 entirely).
>
> **Revised order:** Phase 1 (AI works) → Phase 2 (security) **+ Phase 5a (Supabase
> storage/auth)** → Phase 3 (tests) → Phase 4 (deploy). A private single-author preview
> can deploy on ephemeral storage in between, but no public launch until storage persists.
